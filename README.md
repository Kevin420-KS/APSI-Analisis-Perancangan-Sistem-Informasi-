1. **DFD (Data Flow Diagram)**
   DFD adalah diagram yang menggambarkan aliran data dalam suatu sistem, mulai dari input yang diterima hingga output yang dihasilkan. DFD menampilkan proses, data store, entitas eksternal, dan bagaimana data berpindah di antara komponen-komponen tersebut. DFD tidak fokus pada urutan proses atau logika, melainkan pada **alur data dan transformasinya** dalam sistem.

2. **ERD (Entity Relationship Diagram)**
   ERD digunakan untuk memodelkan struktur data dan hubungan antar entitas dalam sistem basis data. Diagram ini menampilkan entitas (tabel), atribut (kolom), serta hubungan antar entitas seperti one-to-many atau many-to-many. ERD sangat berguna dalam tahap perancangan basis data untuk menggambarkan bagaimana data akan disimpan dan terhubung.

3. **Use Case Diagram**
   Use case diagram menjelaskan interaksi antara aktor (pengguna atau sistem lain) dengan sistem yang dirancang. Diagram ini menunjukkan fungsi-fungsi utama (use case) yang tersedia bagi aktor dan menggambarkan bagaimana pengguna dapat menggunakan sistem. Use case diagram sangat berguna untuk memahami kebutuhan fungsional sistem dari perspektif pengguna.

4. **Activity Diagram**
   Activity diagram menggambarkan alur kerja atau urutan aktivitas dalam sistem, termasuk percabangan, keputusan, dan kondisi. Diagram ini cocok untuk menjelaskan **alur logika proses**, seperti bagaimana pengguna melakukan login, registrasi, hingga mendapatkan informasi. Activity diagram fokus pada urutan langkah yang terjadi dalam suatu proses.

5. **Class Diagram**
   Class diagram digunakan untuk merepresentasikan struktur kelas dalam pemrograman berorientasi objek. Diagram ini menampilkan kelas, atribut, metode (fungsi), dan hubungan antar kelas (seperti asosiasi, pewarisan, atau dependensi). Class diagram membantu memahami struktur kode dan bagaimana objek saling berinteraksi.

6. **Sequence Diagram**
   Sequence diagram menjelaskan **urutan interaksi** antar objek dalam sistem dari waktu ke waktu. Diagram ini menunjukkan bagaimana objek saling berkomunikasi melalui pesan (message) dalam suatu skenario. Sequence diagram sangat berguna untuk menggambarkan **alur eksekusi sistem secara dinamis**, seperti alur login atau permintaan data.

7. **Perbedaan ERD dan Class Diagram**
   Meskipun sekilas mirip karena sama-sama menampilkan entitas dan atribut, ERD dan class diagram berbeda dalam tujuan dan konteks penggunaannya.

   * **ERD** digunakan dalam konteks **basis data relasional**, berfokus pada struktur tabel dan hubungan antar data. Tidak mencakup fungsi atau logika.
   * **Class diagram** digunakan dalam **pemrograman berorientasi objek**, menampilkan **atribut dan metode**, serta hubungan antar kelas seperti pewarisan dan asosiasi. Class diagram mencakup lebih banyak aspek perilaku sistem dibandingkan ERD.
